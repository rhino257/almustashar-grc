## 1. الغرض

تطبيقاً لمبدأ «الأدوار لا الأشخاص»، تربط هذه الوثيقة كل **دورٍ وظيفي** بشاغله الحالي، وتوزّع المسؤوليات على الأنشطة الحوكمية عبر مصفوفة RACI. وهي الوثيقة المُكمّلة للقسم الرابع من ميثاق الامتثال والحوكمة (GOV-CHARTER-001).

## 2. ربط الأدوار بالأشخاص

| الدور الوظيفي | الرمز | الشاغل الحالي |
|---|---|---|
| مدير المشروع (المدير العام) | PM | أسامة عارف السقاف |
| راعي المشروع / رئيس الحوكمة | GRC | محمد عبدالرحمن أحمد الأبيض |
| مسؤول أمن المعلومات | CISO | محمد عبدالرحمن أحمد الأبيض |
| مسؤول حماية البيانات | DPO | محمد (مؤقتاً) |
| مالك النظام/المنتج | PO | يُحدّد |
| مسؤول التطوير الآمن | SecDev | يُحدّد |
| مسؤول العمليات والبنية | Ops | يُحدّد |
| الشريك الداعم (البنك) | BANK | نبيل علي المدعي |

## 3. دليل رموز RACI

- **R (Responsible) — المُنفّذ:** مَن يقوم بالعمل فعلياً.
- **A (Accountable) — المُساءَل:** صاحب القرار النهائي والمسؤولية (واحدٌ لكل نشاط).
- **C (Consulted) — المُستشار:** يُؤخذ رأيه قبل التنفيذ (حوارٌ ثنائي).
- **I (Informed) — المُبلّغ:** يُعلم بالنتيجة (اتجاهٌ واحد).

## 4. مصفوفة RACI

| النشاط | PM | GRC | CISO | DPO | PO | SecDev | Ops | BANK |
|---|---|---|---|---|---|---|---|---|
| اعتماد الميثاق والسياسات | A | R | C | C | C | C | C | I |
| إدارة المخاطر (تقييم/معالجة) | I | A | R | C | C | C | C | I |
| الامتثال لحماية البيانات (PDPL) | I | C | C | A/R | C | C | C | I |
| أمن المعلومات التشغيلي (ISMS) | I | C | A/R | C | C | C | R | I |
| التطوير الآمن | I | I | A | C | C | R | C | — |
| تشغيل البنية On-Prem والنسخ الاحتياطي | I | I | C | — | — | C | A/R | C |
| حوكمة الذكاء الاصطناعي و guardrails | I | A | C | C | C | R | C | I |
| الاستجابة للحوادث | I | I | A | C | — | C | R | I |
| التقارير الفنية الدورية للبنك | A | R | C | C | C | C | C | I |
| القرارات الاستراتيجية (التوسّع) | A* | C | C | — | C | — | — | A* |

> \* القرارات الاستراتيجية (كالتوسّع لأسواقٍ جديدة) تتطلّب **اتفاقاً كتابياً مسبقاً بين الطرفين** (بند ثامناً/2 من العقد)، فالمساءلة فيها **مشتركة** بين مدير المشروع (الطرف الأول) والبنك (الطرف الثاني).

## 5. ملاحظة فصل المهام (Segregation of Duties)

يجمع شاغلٌ واحدٌ حالياً أدوار GRC و CISO و DPO. وهذا **استثناءٌ مؤقتٌ مقبولٌ** في المرحلة المبكرة، لكنه مُسجّلٌ صراحةً بوصفه انحرافاً عن مبدأ فصل المهام (ISO/IEC 27001 — الضابط A.5.3)، مع خطةٍ لفصل دور DPO مستقبلاً لضمان استقلاليته الرقابية.
