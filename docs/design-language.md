# لغة التصميم وواجهة المستخدم (Design Language)

## 1. الفلسفة: نحن «أداةٌ موثوقة»، لا «ماسنجر»

واجهة محادثة الذكاء الاصطناعي **ليست تطبيق مراسلة ولا روبوت دردشة تقليدي**. هي سطحٌ يقع بين
محرّك بحثٍ ومحرّر نصوصٍ ومحادثة. من يعاملها كـ«مراسلة» يحصل على فقاعاتٍ لطيفةٍ وثقةٍ مكسورة؛
ومن يعاملها كـ«صفحة نتائج» يحصل على جدارٍ من النصّ بلا ذاكرة. نموذجنا الصحيح: **أداةٌ
حواريةٌ جادّةٌ تعرض مصادرها وتُفصح عن يقينها** — وهذا ينسجم مع طبيعتنا القانونية.

أعمدة فلسفتنا الأربعة (مقتبسة من أقوى أنماط 2026):

| العمود | المعنى عندنا |
| --- | --- |
| شفافية القدرة | يعرف المستخدم ماذا نفعل وما لا نفعله قبل أن يكتب |
| أنماط التعافي | ماذا يحدث عند فشل التوليد؟ خطأٌ واضحٌ + إعادة محاولة |
| إظهار الثقة | شارة الإسناد + المصادر + إخلاء المسؤولية = «الإيصالات» |
| إتاحة الوصول | يعمل على الجوال وقارئ الشاشة وبتباينٍ كافٍ |

## 2. المرجع: لغة تصميم ChatGPT وكيف نربطها بنا

| عنصر ChatGPT/Claude | قرارنا |
| --- | --- |
| عمودٌ واحدٌ مركزيٌّ بعرضٍ محدود (~720–768px) | نعتمده؛ نقصر عمود المحادثة على 720px |
| رسائل **بعرضٍ كامل مسطّحة** لا فقاعات SMS | نعتمده لرسائل المساعد (إطار «أداة») |
| Markdown غنيّ للردّ (عناوين/قوائم/جداول/كود) | نعتمده — الردّ القانوني يحتاج بنية |
| مؤشّر «حيّ» (caret) أثناء البثّ | مربّعٌ صغيرٌ أحاديّ اللون (نمط Claude) |
| مُلحِّن (composer) هادئٌ مرسّى أسفل الشاشة | نعتمده، ينمو للأعلى حتى حدٍّ ثمّ يتمرّر |
| استشهادات مرقّمة (نمط Perplexity) | **أساسي** — لأنّنا RAG؛ المصادر «غير قابلة للتفاوض» |
| محفوظات المحادثات في رفٍّ جانبي | لاحقاً (Tier-2)؛ الآن شاشةٌ واحدة |

## 3. مبادئ التصميم الستّة

1. **المحتوى هو البطل** — الإجابة والمصادر في الصدارة؛ الزخرفة تتراجع.
2. **أحاديّة اللون المؤسّسية** — `#121212`/`#f5f4f5` + لمساتٌ دلاليةٌ مكتومة فقط.
3. **الثقة قبل الجمال** — الإسناد وإخلاء المسؤولية بارزان دائماً.
4. **اتّساقٌ هندسي** — شبكة 8px، زوايا معتدلة (10–14px)، ظلالٌ خفيفةٌ جداً.
5. **حركةٌ هادفة** — نبضةٌ للمرحلة النشطة ووميض caret فقط؛ لا بهرجة.
6. **عربيٌّ أوّلاً (RTL-first)** — لا «إنجليزيٌّ مقلوب»؛ طباعةٌ عربيةٌ أصيلة.

## 4. اللوحة اللونية (Color Tokens)

| الرمز | فاتح | داكن | الاستخدام |
| --- | --- | --- | --- |
| brandBlack | `#121212` | خلفية | النصّ/السطح الأساسي |
| brandOffWhite | `#F5F4F5` | نصّ | الخلفية/النصّ المعكوس |
| surface | `#FFFFFF` | `#1C1C1C` | سطح الرسائل |
| surfaceHi | `#F2F2F2` | `#262626` | خلفية رسالة المساعد |
| hairline | `#E0E0E0` | `#3A3A3A` | الفواصل والحدود |
| grounded | `#1F7A4D` | `#4CAF7D` | شارة «مسند» |
| ungrounded | `#B23A3A` | `#E57373` | شارة «غير مسند»/الخطأ |
| warning | `#8A6D00` | `#D4A72C` | تحذير/قيد التحقّق |

> **الوضع الافتراضي:** `ThemeMode.system`. الهوية dark-first، لكن نحترم تفضيل النظام.

## 5. الطباعة العربية (الأهمّ — وأكثر ما يرفع الجودة المُدرَكة)

العربية ليست لاتينيةً مقلوبة. القواعد الإلزامية:

| المعيار | القيمة عندنا | السبب |
| --- | --- | --- |
| الخطّ | IBM Plex Sans Arabic (OFL) | مصمّمٌ لقراءة الشاشة، ثنائيّ اللغة |
| ارتفاع السطر (body) | **1.75** | حروف العربية تمتدّ فوق/تحت الخطّ + التشكيل يحتاج تنفّساً |
| ارتفاع السطر (عناوين) | 1.35 | تنفّسٌ أقلّ للعناوين |
| حجم المتن | 16–17px (الجوال ≥16px) | أقلّ من 16px يُتعب ويُفعّل تكبير iOS |
| تباعد الأحرف | **0 دائماً** | العربية متّصلة؛ التباعد يكسر الروابط |
| الأرقام | عربية غربية (0–9) افتراضاً | اتّساق وقابلية قراءة |
| المحاذاة | `text-align: start` لا `right` | لا تنكسر عند كتابة أحرفٍ لاتينية |

سُلّم طباعيٌّ مقترح (Flutter `TextTheme`):

​
// داخل AppTheme، أضِف textTheme مضبوطاً للعربية:
static TextTheme _arabicText(Color onSurface) => TextTheme(
headlineSmall: TextStyle(fontSize: 22, height: 1.35, fontWeight: FontWeight.w600, letterSpacing: 0, color: onSurface),
titleLarge:    TextStyle(fontSize: 20, height: 1.40, fontWeight: FontWeight.w600, letterSpacing: 0, color: onSurface),
bodyLarge:     TextStyle(fontSize: 17, height: 1.75, fontWeight: FontWeight.w400, letterSpacing: 0, color: onSurface),
bodyMedium:    TextStyle(fontSize: 16, height: 1.75, fontWeight: FontWeight.w400, letterSpacing: 0, color: onSurface),
labelLarge:    TextStyle(fontSize: 14, height: 1.60, fontWeight: FontWeight.w500, letterSpacing: 0, color: onSurface),
);
// ملاحظة: مرّر textTheme: _arabicText(onSurface) داخل ThemeData.

## 6. التخطيط (Layout)

| العنصر | القاعدة |
| --- | --- |
| عمود المحادثة | عرضٌ أقصى **720px** مركزيّ؛ هامش 16px على الجوال |
| رسالة المساعد | **عرضٌ كامل مسطّح** بخلفية `surfaceHi`، بلا فقاعة SMS |
| رسالة المستخدم | مُحتواةٌ يميناً، تعبئةٌ خفيفة، عرضٌ أقصى ~85% |
| المُلحِّن (composer) | مرسّى أسفل الشاشة (لا عائم)، ينمو حتى ~6 أسطر ثمّ يتمرّر |
| زرّ الإيقاف | قرب المُلحِّن، ≥44px للمسة الإبهام |
| طول السطر | 65–80 محرفاً للقراءة المستدامة |

> **مكافحة خطأ شائع:** المُلحِّن لا يطفو فوق آخر رسالة؛ أعطِ التدفّق هامشاً سفلياً = ارتفاع المُلحِّن.

## 7. تشريح الواجهة (Anatomy)

​
+---------------------------------------------------+
AppBar: العنوان + زرّ محادثة جديدة
+---------------------------------------------------+
بانر إخلاء المسؤولية (دائم، خفيف)
+---------------------------------------------------+
[تدفّق الرسائل — عمود 720px مركزي]
  • رسالة المستخدم (يمين، محتواة)
  • رسالة المساعد (عرض كامل):
  • نصّ Markdown + caret أثناء البثّ
  • شارة الإسناد
  • صندوق المصادر (قابل للطيّ) + تبويب النسخ
  • مؤشّر المراحل (عائم فوق المُلحِّن أثناء العمل)
+---------------------------------------------------+
المُلحِّن: حقل + إرسال/إيقاف
+---------------------------------------------------+

## 8. حالات رسالة المساعد الستّ (مربوطة بعقد SSE)

| الحالة | حدث SSE | المظهر |
| --- | --- | --- |
| في الانتظار (Queued) | `QueuedEvent` | سطرٌ متلألئ + «في قائمة الانتظار (رقمك N)» |
| التحليل/البحث/إعادة الترتيب/التوليد | `StatusEvent` | مؤشّر المراحل النابض (4 مراحل) |
| البثّ (Streaming) | `TokenEvent` | نصٌّ يتدفّق + caret مربّع وامض |
| مكتملة (Complete) | `DoneEvent` | نصٌّ نهائي + شارة الإسناد + المصادر + زرّ نسخ |
| خطأ (Error) | `ErrorEvent` | رسالة خطأ صريحة + «إعادة المحاولة» |
| الخدمة مزدحمة | `429` | بانر «الخدمة مزدحمة» + Retry-After |

> معظم الفِرق تشحن حالتين فقط (بثّ/تمّ) فتبدو الواجهة «متقطّعة». نحن نصمّم الستّ.

## 9. نمط البثّ (Streaming) — حيث تلتقي الهندسة بالتصميم

| العنصر | القاعدة |
| --- | --- |
| رسالة المستخدم التفاؤلية | اعرضها فوراً قبل مغادرة الطلب (يوفّر 100–300ms مُدرَكة) |
| أوّل رمز (first-token) | الهدف < 800ms؛ وإلّا اعرض حالة «في الانتظار» |
| مؤشّر caret | مربّعٌ صغيرٌ وامض؛ إسقاطه يجعل البثّ المتوقّف يبدو مكتملاً |
| تجميع الرموز | ادفع كلّ 30–60ms لا كلّ رمز (أنعم، أقلّ استهلاكاً) |
| التمرير التلقائي | فقط إذا كان العرض ضمن 100px من الأسفل؛ لا تسحب المستخدم لأسفل إن كان يقرأ |

> **ملاحظة واقعية:** ميزانيتنا الحالية TTFT = 1–2s أعلى من 800ms المثالية؛ لذا حالة «في الانتظار»
> النابضة **إلزامية** لسدّ الفجوة المُدرَكة حتى يبدأ البثّ.

## 10. الثقة والاستشهادات (تخصّصنا)

- **شارة الإسناد:** «مسنَد بالمصادر» (أخضر) / «غير مسنَد» (أحمر) / «قيد التحقّق» (محايد)، مشتقّةٌ من `is_grounded`.
- **صندوق المصادر:** قابلٌ للطيّ، يعرض المواد القانونية مرقّمةً (اسم القانون + رقم المادة + مقتطف + درجة).
- **تبويب النسخ:** «النصّ الأصلي» ⇄ «تعديل صنعاء» (`original` / `sanaa_amended`) — يُظهر الفرق بوضوح.
- **بانر إخلاء المسؤولية:** دائمٌ ظاهر — «للاسترشاد فقط وليس استشارةً ملزِمة».
- **لا تزييف:** إن لم توجد مصادر، تُعرض «غير مسنَد» بصدق؛ لا نخترع استشهادات.

## 12. إمكانية الوصول (Accessibility) — حرِجة وكثيراً ما تُهمَل

| البند | التطبيق في Flutter |
| --- | --- |
| منطقة حيّة للبثّ | `Semantics(liveRegion: true)` حول رسالة المساعد |
| تباين الألوان | WCAG AA (≥4.5:1 للنصّ) — ألواننا الدلالية مضبوطة |
| حدّ أدنى للخطّ | ≥16px للعربية على الجوال |
| تقليل الحركة | احترم `MediaQuery.disableAnimations` — أوقف النبض والوميض |
| لوحة المفاتيح (الويب) | مسارٌ لكلّ إجراء؛ Enter للإرسال، Esc لإلغاء التحرير |
| إدارة التركيز | يبقى في المُلحِّن بعد الإرسال؛ لا تسرق التركيز عند اكتمال الردّ |

## 13. خصوصيات RTL

- `Directionality(textDirection: rtl)` على مستوى الجسم.
- استخدم `EdgeInsetsDirectional` و`AlignmentDirectional` لا `left/right`.
- `textAlign: TextAlign.start` لا `right`.
- اعكس الأيقونات الاتجاهية (سهم الإرسال، الرجوع) تلقائياً.
- الأرقام الغربية (0–9) افتراضاً.

## 14. الحركة (Motion)

| العنصر | الحركة |
| --- | --- |
| caret | وميضٌ هادئٌ (~1s) |
| المرحلة النشطة | نبضةٌ خفيفةٌ (scale/opacity) |
| ظهور المصادر | انزلاقٌ/تلاشٍ قصير (<200ms) |
| تقليل الحركة | استبدل كلّ ما سبق بتغيّرٍ فوري |

## 15. قرار الفقاعات (Decision Record)

- **القرار:** رسائل المساعد **بعرضٍ كامل مسطّحة** (إطار «أداة»)، ورسائل المستخدم محتواةٌ يميناً.
- **السبب:** الفقاعات الملوّنة تُوحي بالدردشة العابرة وتُضعف الثقة المطلوبة لتطبيقٍ قانوني.
- **الأثر التقني:** تخصيص `MessageOptions`/`BubbleStyle` في `flutter_gen_ai_chat_ui`
  لإلغاء فقاعة المساعد واستبدالها بحاوية عرضٍ كامل.

## 17. قائمة ما قبل الإطلاق (Pre-launch Checklist)

- [ ] عمود المحادثة ≤720px، 65–80 محرفاً/سطر.
- [ ] رسائل المساعد عرضٌ كامل مسطّح (لا فقاعات SMS).
- [ ] أوّل رمز <800ms أو حالة «في الانتظار» ظاهرة.
- [ ] caret ظاهرٌ أثناء البثّ، يختفي عند الاكتمال.
- [ ] تمرير تلقائيٌّ ضمن 100px من الأسفل فقط.
- [ ] المُلحِّن مرسّى لا عائم.
- [ ] كلّ رسالة مساعدٍ مسنّدةٌ تعرض المصادر؛ غير المسنّدة تعرض الشارة بصدق.
- [ ] إخلاء المسؤولية دائمٌ ظاهر.
- [ ] ارتفاع سطر العربية ≥1.7، خطّ ≥16px، تباعد 0.
- [ ] منطقة حيّة + تقليل الحركة + مسار لوحة مفاتيح.
- [ ] خطأ التوليد يعرض «إعادة المحاولة».

## 18. قرارات مفتوحة

1. إظهار اسم النموذج/أفاتار للمساعد؟ (يميل المعيار لإظهار مصدر الردّ — مؤجّل).
2. الوضع الافتراضي: `system` أم داكنٌ دائماً؟ (الحالي: `system`).
3. رفّ المحفوظات الجانبي (Tier-2).
4. اختبار حجم الخطّ الأمثل لـ IBM Plex Sans Arabic (16 مقابل 17px) تجريبياً.
