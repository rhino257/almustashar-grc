# ADR-0007: الفهرس الأعمى وتجزئة كلمات المرور

- **الحالة:** Accepted
- **التاريخ:** 2026-07-04
- **صلة:** SEC-CRYPTO-001 v0.2، data-model v0.4، ADR-0008

## السياق
نشفّر الهاتف/البريد/البريد الاحتياطي (PII) لكنّنا نحتاج البحث والتفرّد عليها
(unique lookup) دون فكّ التشفير.

## القرار
- **فهرس أعمى (blind index) بـ HMAC-SHA-256** على الهاتف/البريد/البريد الاحتياطي:
  الأعمدة `phone_hash`, `email_hash`, `recovery_email_hash` (فريدة) للبحث والتفرّد،
  مقابل الأعمدة المشفّرة `*_enc`.
- **كلمات المرور بـ Argon2id** (مفعّلة، ليست مؤجّلة).
- رموز التحديث (refresh) تُخزَّن كـ SHA-256.

## النتائج
- (+) بحث/تفرّد على PII دون كشفها.
- (−) HMAC حتمي يسمح بمطابقة تساوٍ فقط (لا بحث جزئي) — مقبول.

## البدائل المرفوضة
- **تخزين PII كنص صريح للبحث:** خطر خصوصية غير مقبول.
- **تجزئة كلمات المرور بـ SHA وحده:** غير كافٍ ضد الكسر بالعتاد.
