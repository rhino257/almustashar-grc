# ADR-0008: تأجيل تشفير الحقول عبر CryptoService

- **الحالة:** Accepted
- **التاريخ:** 2026-07-04
- **صلة:** SEC-CRYPTO-001 v0.2، ADR-0007

## السياق
نريد بنية جاهزة لتشفير الحقول (AES-256-GCM) دون إبطاء بناء الهيكل الأول (MVP).

## القرار
- تجريد `CryptoService` بمزوّدين: **NullCipher** (MVP، بلا تشفير فعلي) و
  **AesGcmCipher** (لاحقاً)، يُبدَّل عبر متغيّرات البيئة `CIPHER_PROVIDER`/`CIPHER_KEY`.
- TLS و Argon2id و HMAC **ليست مؤجّلة** — المؤجّل فقط تشفير الحقول على مستوى الصف.
- بقايا مفتوحة: KMS مقابل env للمفاتيح، تشفير القرص الكامل، وتشفير سرّ TOTP
  (مؤجّل تبعاً لـ ADR-0003).

## النتائج
- (+) طريق ترقية واضح دون إعادة هيكلة (تبديل provider).
- (−) بيانات الحقول غير مشفّرة أثناء MVP (مقبول ضمن بيئة On-Prem محكومة مؤقتاً).

## البدائل المرفوضة
- **تنفيذ AES-GCM كاملاً الآن:** يبطئ الهيكل الأول دون حاجة فورية.
