# ADR-0009: الهيكل السائر (Walking Skeleton) عبر mock repositories

- **الحالة:** Accepted
- **التاريخ:** 2026-07-04
- **صلة:** ADR-0001، ARC-API-001

## السياق
الواجهة جاهزة والعقد معتمد، لكن الخادم غير مبنيّ. نريد تشغيل التطبيق كاملاً
من طرف إلى طرف مبكراً دون انتظار الخادم الحقيقي.

## القرار
- بناء التطبيق فوق **mock repositories** تطابق العقد (`MockChatRepository`،
  `MockAuthRepository` عبر providers).
- بناء backend رفيع (thin FastAPI + LangGraph) كـ walking skeleton، ثم
  **استبدال المزوّد بسطرين** للانتقال من mock إلى التنفيذ الحقيقي.

## النتائج
- (+) تكامل end-to-end مبكر واختبار تجربة المستخدم قبل اكتمال الخادم.
- (+) انتقال منخفض الكلفة إلى التنفيذ الحقيقي.
- (−) الحاجة لإبقاء الـ mocks متوافقة مع العقد حتى الاستبدال.

## البدائل المرفوضة
- **انتظار الخادم الحقيقي قبل ربط الواجهة:** يؤخّر التغذية الراجعة والتكامل.
