# ADR-0001: التطوير بمنهج العقد أولاً (Contract-First)

- **الحالة:** Accepted
- **التاريخ:** 2026-07-04
- **صلة:** ARC-API-001، ADR-0009

## السياق
نطوّر واجهة Flutter وخادم FastAPI بالتوازي. بدون مصدر حقيقة موحّد لواجهة الـ API
يتفرّع التطبيق والخادم ويصعب دمجهما لاحقاً.

## القرار
نعتمد **العقد أولاً**: يُكتب ويُعتمد `openapi.yaml` + `api-contract.md` **قبل**
مواءمة التطبيق أو بناء الخادم. كلاهما يلتزم بالعقد كمصدر حقيقة واحد.

## النتائج
- (+) تكامل متوقّع، وتوليد نماذج/عملاء من العقد ممكن.
- (+) يمكن تطوير الطرفين بالتوازي على mocks تطابق العقد.
- (−) أي تغيير في الواجهة يتطلّب تحديث العقد أولاً (انضباط إضافي).

## البدائل المرفوضة
- **الكود أولاً ثم توليد العقد:** يؤخّر اكتشاف عدم التوافق ويجعل العقد تابعاً لا حاكماً.
