# معيار التشفير

## 1. الغرض والنطاق
يحدّد هذا المعيار الخوارزميات وأطوال المفاتيح وأنماط التشغيل المسموحة
والمحظورة لحماية البيانات أثناء النقل وعند التخزين، وإدارة المفاتيح، في
تطبيق المستشار القانوني الذكي.

## 2. التشفير أثناء النقل (Data in Transit)
- TLS 1.3 (مفضّل)؛ TLS 1.2 حدٌّ أدنى. ما دونه محظور.
- HSTS مفعّل؛ إعادة توجيه HTTP إلى HTTPS.
​
TLS 1.3 (preferred):
TLS_AES_256_GCM_SHA384
TLS_CHACHA20_POLY1305_SHA256
TLS_AES_128_GCM_SHA256
TLS 1.2 (minimum):
ECDHE-ECDSA-AES256-GCM-SHA384
ECDHE-RSA-AES256-GCM-SHA384
Disabled: TLS 1.0/1.1, SSLv3, RC4, 3DES, static-RSA, CBC-only suites
> ملاحظة حوكمة: تشفير النقل **لا يُؤجَّل**؛ تكلفته شبه معدومة وهو خط الدفاع
> الأول، خلافاً لتأجيل تشفير التخزين (القسم 9).

## 3. التشفير عند التخزين (Data at Rest)
- الخوارزمية المعتمَدة: **AES-256-GCM** (authenticated encryption).
- التشفير على مستوى الحقل للبيانات الحسّاسة عبر `CryptoService` (القسم 7).
- تشفير القرص/الوحدة (volume encryption) للخادم موصى به كطبقةٍ إضافية.

## 4. التجزئة (Hashing)
| الغرض | الخوارزمية المعتمَدة |
| --- | --- |
| سلامة البيانات / البصمات | SHA-256 أو SHA-384 |
| الأسرار قصيرة العمر (OTP) | **Argon2id** + TTL قصير + تحديد معدّل |
| كلمات المرور (مستقبلاً) | **Argon2id** (أو bcrypt كحدٍّ أدنى) |
| رموز المصادقة (HMAC) | HMAC-SHA-256 |

> تصحيح: لا يُستخدم SHA المجرّد لتخزين OTP؛ يُكسر بالقوة الغاشمة لقصره.

## 5. التشفير غير المتماثل والتواقيع
| الغرض | المعتمَد |
| --- | --- |
| RSA | مفتاح >= 3072 بت |
| ECDSA/ECDH | منحنى P-256 أو P-384 |
| التواقيع الحديثة | Ed25519 |

## 6. توليد العشوائية
- مولّد عشوائيٍّ آمنٌ تشفيرياً (CSPRNG) حصراً لكل المفاتيح والـ nonces
  وقيم OTP. يُمنع استخدام مولّدات عامة (مثل `random`).

## 7. تطبيق CryptoService (التصميم القابل للتشفير)
- واجهةٌ مجرّدة `Cipher { encrypt(), decrypt() }`.
- التطبيق الحالي: `NullCipher` (تمرير بلا تشفير) — تأجيلٌ مُسجَّل (القسم 9).
- التطبيق المستقبلي: `AesGcmCipher` (AES-256-GCM) دون أي ترحيلٍ للمخطّط
  (الأعمدة `TEXT` مهيّأةٌ لحجم النص المشفّر).
​
provider switch via env: CIPHER_PROVIDER = null | aes_gcm
key via env/KMS:         CIPHER_KEY = base64(32 bytes)   # required for aes_gcm
- `users.phone_enc`: تشفيرٌ قابلٌ للفكّ (admin-decryptable) لخدمة العملاء.

## 8. إدارة المفاتيح (Key Management)
- توليد المفاتيح بـ CSPRNG وبالأطوال أعلاه.
- التخزين: مدير أسرار/KMS؛ في الـ MVP تُحقن عبر بيئةٍ محميةٍ خارج Git.
- الفصل: مفاتيح مختلفة لكل بيئة (dev/prod)؛ لا مفاتيح في الكود إطلاقاً.
- الدوران (rotation): دوريٌّ وعند الاشتباه بالتسريب؛ دعم إصدارات المفاتيح
  (key versioning) لفكّ النصوص القديمة.

## 9. الامتثال والاستثناءات الحالية
- **استثناء مُسجَّل:** تأجيل تشفير التخزين في الـ MVP مع إلزام أن يكون
  الكود «قابلاً للتشفير» (القسم 7). يُغلق الاستثناء عند تفعيل `AesGcmCipher`.
- تشفير النقل (TLS) **غير مشمولٍ بالتأجيل** ويبقى إلزامياً.

## 10. المحظورات
محظورٌ صراحةً: MD5، SHA-1، DES/3DES، RC4، نمط ECB، مفاتيح مضمّنة في
الكود (hardcoded)، وتطوير خوارزميات تشفيرٍ ذاتية (roll-your-own crypto).
